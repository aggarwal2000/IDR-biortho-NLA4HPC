IDR(s)-biortho CUDA-C++ Implementation
Requirements and Usage

Requirements:

  • C++11 compliant compiler
  • CUDA 10.2 or higher
  • cmake 3.14.6 or higher

Usage:

Building:

Go to folder containing the project code and do:

mkdir build_dir
cd build_dir
cmake ..
cmake --build .

This produces executable biorthoIDR

Running Application:

./biorthoIDR [path to matrix market file for matrix A(in coordinate format)] [Optional-- path to matrix market file for vector b(in array format)]

The second argument is optional. Vector b is generated by the program itself, in case it is not provided.

Changing few parameters:

We can set the value of initial guess, solver parametrs such as atol(absolute tolernace), rtol(relative tolerance), max_iter(maximum number of iterations allowed) and value of s(shadow space number) and type of preconditioner by editing the main(int,char*[]) function in main.cu file.

Following is the code snippet from the main(int,char*[]) function :

Dense_Matrix x(b->GetRows(), 1, b->GetRows(), ORDER::COLUMN_MAJOR,
CPU_EXISTENCE::EXISTENT, GPU_EXISTENCE::EXISTENT);
for (int i = 0; i < x.GetRows(); i++) //Set initial guess
{
x.GetCPUValues()[i].x = 0;
x.GetCPUValues()[i].y = 0;
}
x.CopyMatrix_cpu_to_gpu();
//Generate preconditioner - Richardson/Jacobi(Only 2 types are there)
Preconditioner* precond = Generate_Preconditioner(PRECONDITIONER_TYPE::RICHARDSON, *A);
Solver solver_obj;
//Can set atol, rtol, max_iter etc... using solver_obj setter functions
solver_obj.SetRtol(1e-04);
solver_obj.SetAtol(1e-12);
solver_obj.SetMax_iter(100);
int shadow_space_number = 8;
solver_obj.PIDR_Solver(*A, *b, x, *precond, shadow_space_number); //Can change value of shadow space number here

Output:

All the solver parametrs such as total iterations, final true residual, final iteratively computed residual, runtime in milliseconds, sparse-matrix vector multipliation count etc. and most importantly the conclusion about convergence are printed on the console.

It also generates a log file:

"log.txt" : contains iteratively computed residuals and the timings when they are computed.

In case of successful convergence, following matrix market files are generated:

"Vector_x.mtx" : contains solution vector x

"Vector_res.mtx" : contains vector of iteratively computed residuals
